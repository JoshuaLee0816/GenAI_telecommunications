# Output JSON Schema (Desing Document)

This document defines the strcture of the output JSON generated by the output JSON generated by the GenAI workflow for support ticket processing.

## JSON Fields

### ticket_type
Purpose: Classify the ticket into a predifined category (e.g., network_outage, billing_issue, equipment_repair)
Type: string
Possible values: network_outage, billing_issue, equipment_repair, other
Derived from: issue_summary, issue_description, service_type
Used by downstream: Yes, for routing and workflow automation

### severity
Purpose: Assess the urgency and impact level of the ticket.
Type: string
Possible values: low, medium, high, critical
Derived from: historical_ticket_count, outage_flag, description, device_type
Used by downstream: Yes, for priortization

### escalation_required
Purpose: Indicates whether the ticket should be escaleted to a higher support levle.
Type: boolean
Derived from: severity, service_type, rules in knowledge base
Used by downstream: Yes, triggers automated escalation workflow

### recommendation actions
Purpose: Suggest a sequece of steps to resolve the ticket.
Type: array of strings
Example ["Check base station status", "notify on-call engineer"]
Derived from: ticket_type, knowledge base, network_type
Used by downstream: Yes, can be consumed by automation or operator.

### confidence
Purpose: Represents the system's confidence in its classification and recommendations.
Type: double number (0.0 - 1.0)
Derived from: internal model probability or heuristic scoring
Used by downstream: Yes, for fallback or human review if confidence is low
